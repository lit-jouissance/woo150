# 기출 → bank.json 추출 가이드

원본 기출 이미지는 사이트에 넣지 않습니다. **이미지에서 정보만 뽑아** 아래 형식의 가벼운 JSON으로 저장하고, 사이트는 그 JSON(`/data/bank.json`)만 불러옵니다.

---

## 1. JSON 스키마 (한 문항 = 객체 하나)

`bank.json`은 문항 객체들의 **배열**입니다. 새 시험이 나오면 객체를 뒤에 **추가(append)**만 하면 됩니다.

공통 필드
- `id` (문자열, 고유): 예 `"2026-mat-12"` (연도-과목-번호)
- `subject` (문자열): `kor` | `mat` | `engR` | `his` | `soc` | `eth`
- `unit` (문자열): 앱의 단원명과 일치시키면 빈도 분석이 정확해짐 (아래 표 참고)
- `source` (문자열): 출처 표기, 예 `"2026 수능 12번"`
- `question` (문자열): 문제 (수학은 LaTeX 인라인 `$...$`)
- `choices` (배열, 5개): `["① ...","② ...","③ ...","④ ...","⑤ ..."]`. 단답형이면 `null`
- `answer` (숫자 1~5, 또는 단답 문자열)

과목별 추가 필드
- 수학(mat): `hint1`, `hint2`, `solution`(단계별 상세 풀이, LaTeX)
- 한국사·생윤·사문(his/soc/eth): `concept`(개념 찾아보기용), `explanation`
- 국어·영어독해(kor/engR): `passage`(없으면 빈 문자열), `explanation`

### 단원명(unit) 권장값
- kor: 독서-인문·예술 / 독서-사회·문화 / 독서-과학·기술 / 독서-주제통합 / 문학-현대시 / 문학-현대소설 / 문학-고전시가 / 문학-고전산문 / 화법과 작문
- mat: 수학Ⅰ-지수와 로그 / 수학Ⅰ-삼각함수 / 수학Ⅰ-수열 / 수학Ⅱ-극한과 연속 / 수학Ⅱ-미분 / 수학Ⅱ-적분 / 확통-경우의 수 / 확통-확률 / 확통-통계
- engR: 대의 파악 / 세부·자료 / 어법·어휘 / 빈칸 추론 / 순서·삽입 / 문단 요약 / 장문
- his: 전근대(선사~조선) / 근대(개항~1910) / 일제강점기 / 현대
- soc: 사회·문화 현상의 탐구 / 개인과 사회 구조 / 문화와 일상생활 / 사회 계층과 불평등 / 현대의 사회 변동 / 도표·통계 분석
- eth: 실천 윤리의 기초 / 생명과 윤리 / 사회와 윤리 / 과학과 윤리 / 문화와 윤리 / 평화와 공존의 윤리

## 2. 비전 추출 프롬프트 (회분별로 사용)

기출 PDF/이미지를 올린 뒤 아래 프롬프트를 함께 주면 위 스키마대로 JSON이 나옵니다.

```
첨부한 수능 기출(영어 듣기 제외)을 읽고, 각 문항을 아래 JSON 스키마의 배열로만 출력하세요.
코드펜스·설명 없이 JSON 배열만. 수식은 LaTeX 인라인 $...$.
도표·그림이 본질인 문항은 "question"에 그림 내용을 글로 충분히 묘사하고 "needsFigure": true 를 추가하세요.

스키마:
{
  "id": "연도-과목-번호",
  "subject": "kor|mat|engR|his|soc|eth",
  "unit": "(권장 단원명)",
  "source": "2026 수능 N번",
  "question": "...",
  "choices": ["① ...","② ...","③ ...","④ ...","⑤ ..."],   // 단답이면 null
  "answer": 1~5 또는 "단답",
  // 수학: "hint1","hint2","solution"
  // his/soc/eth: "concept","explanation"
  // kor/engR: "passage","explanation"
}
```

## 3. 데이터 최신 유지 워크플로

1. 새 모의평가·기출이 나오면 그 **회분만** 비전으로 추출 → JSON 배열 획득
2. `data/bank.json` 배열 끝에 **추가**
3. GitHub에 push → Netlify 자동 재배포 → 사이트 데이터 갱신 완료

> 도표·그림이 꼭 필요한 소수 문항만 이미지를 별도 저장소(GitHub의 작은 이미지 또는 Cloudflare R2)에 올려 URL로 참조하면, 전체 용량은 계속 가볍게 유지됩니다.
